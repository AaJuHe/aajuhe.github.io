Site vs. App

It appears that the main difference between a website and a web application is that a website is primarily used for providing information for the user whereas a web application allows the user to interact with the content on the web page. In the end, it depends on what the person or company prefers to be displayed over the Internet. The idea that would most likely be the best bet is some sort of hybrid between a website and application, which would thus provide the best of both worlds.

An example of a website would be the Fresno Bee, which allows the user to read the Fresno Bee online instead of a physical copy of the newspaper.

An example of a web application would be Mint, which is a site used to create a budget. The user would input the relevant information and have a good idea to see where his/her money is going as well as have an understanding of what his/her net worth is.

Build It and They Will Come

This was something I really had to sit down and think about because I have a few ideas of what kind of website I would create if I were to go in that direction. In the end, I decided if I were to create a website, it would be about me and the things I am interested in (computers, math, music, etc.). I think this would be the best way instead of creating multiple websites. While the site would be mainly about my interests, I would include some tips and advice on said interests for others to use. For example, if someone was struggling in math, I would place some tutorials on how to solve problems and so on. It would start as a website, then eventually it would gradually become a web application and involve more user interaction, which would include inputting problems and asking questions about certain topics.

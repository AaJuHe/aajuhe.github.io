Front-End Frameworks

By Aaron Hearne

Select Your Editor Web Hosting Chrome DevTools Front-End Frameworks Semantic Versioning

I decided to go with UIKit for the Front-End Framework. The biggest advantage UIKit has is it is highly customizable, with over thirty modular components. In addition, most of the components can be combined with one another to create other things. Another thing UIKit has going for it is that it supports both the LESS and Sass CSS preprocessors; if someone is comfortable with using one preprocessor, that person has the option to use that preprocessor instead of the other. With additional features such as navigations and JavaScript components, UIKit is an extremely versatile and popular front-end framework.

Even with all of these features, UIKit is not infallible by any means. If there is one major downside, it is that because it is still fairly new, there is not too many resources in case someone has questions about anything UIKit-related. Because of this, UIKit primarily favors developers who have some experience with frameworks. Outside of that, I believe UIKit is still a framework worth considering.
